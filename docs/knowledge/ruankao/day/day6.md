## 三、今天必须先了解的信息

### 1. 为什么成本管理是高频失分模块

很多考生在成本管理上会出现三类问题：

- 概念会一点，但估算、预算、控制之间的边界不稳。
- 公式背过，但 PV、EV、AC 一放到题目里就容易看错。
- 计算能算出结果，却不会把结果翻译成“项目当前处于什么状态、应采取什么措施”。

所以 day6 的目标不是机械背公式，而是把下面这个问题答完整：

**项目花了多少钱、值回了多少工作量、偏差在哪里、后面怎么办。**

## 四、今天要掌握的核心知识

## 1. 成本管理主线

今天必须记住的主线是：

1. 规划成本管理
2. 估算成本
3. 制定预算
4. 控制成本

### 1. 这条主线怎么理解

可以把它理解为下面这条链：

- 先确定怎么管成本
- 再估算每项工作大概要花多少钱
- 再把各项估算汇总为成本基准和预算
- 最后在执行中监控偏差并纠偏

### 2. 考试中的高频考法

考试不会只问你“流程顺序是什么”，还会把流程和场景绑在一起考：

- 范围变了，但预算没调，属于什么问题
- 没有形成成本基准，后续为什么难以控制
- 只看实际支出，不看挣值，为什么判断不完整

## 2. 成本估算、成本预算、成本控制的区别

### 1. 成本估算

成本估算强调的是：

- 完成某项工作需要多少资金或资源成本
- 它更偏“预测”和“测算”

### 2. 制定预算

制定预算强调的是：

- 把各项成本估算进行汇总
- 分配到控制账户、阶段或时间段
- 形成可用于比较和控制的成本基准

### 3. 控制成本

控制成本强调的是：

- 把实际情况与成本基准进行比较
- 发现偏差、分析原因、采取措施
- 防止未经批准的成本变动失控

### 4. 快速区分法

- 估算：先算大概要花多少
- 预算：再把钱按结构和时间分好
- 控制：执行中持续盯偏差和纠偏

## 3. 常见成本估算方法

### 1. 类比估算

根据历史类似项目进行估算。

特点：

- 速度快
- 成本低
- 精度通常一般

### 2. 参数估算

根据单价、数量、生产率等参数进行估算。

例如：

- 每个门禁点位平均实施成本 1.5 万元
- 本次共 80 个点位
- 则点位建设成本可按参数进行测算

### 3. 自下而上估算

从工作包或活动层级逐项估算，再向上汇总。

特点：

- 工作量大
- 过程更细
- 精度通常较高

### 4. 三点估算

通过最乐观、最可能、最悲观成本进行估算。

常见期望值公式：

- 期望成本 = (最乐观成本 + 4 × 最可能成本 + 最悲观成本) / 6

### 5. 专家判断

当历史数据不完整、场景复杂或技术特殊时，通常需要借助专家经验。

## 4. 成本基准、项目预算与储备

### 1. 成本基准是什么

成本基准是项目经批准、可用于后续对比控制的时间分段预算。

它的价值在于：

- 让项目后续有比较对象
- 让偏差分析有依据
- 让变更和纠偏有落点

### 2. 项目预算是什么

项目预算通常是在成本基准基础上，再加上管理储备等形成的总体资金安排。

简单理解：

- 成本基准更偏“控制基准”
- 项目预算更偏“项目总资金安排”

### 3. 应急储备和管理储备

#### 1. 应急储备

- 用于已识别风险
- 通常属于成本基准的一部分
- 对应“已知的未知”

#### 2. 管理储备

- 用于未识别风险或管理层预留
- 通常不计入成本基准
- 更接近“未知的未知”

### 4. 高频易混点

- 应急储备不等于管理储备
- 成本基准不等于项目预算
- 预算不是拍脑袋决定，而是基于估算和批准形成

## 5. 挣值管理的三个基础量

### 1. PV（Planned Value，计划价值）

PV（Planned Value，计划价值）表示截至某一时点，按计划本应完成工作的预算价值。

可以简单理解为：

- 按计划本来应该挣到多少“预算价值”

### 2. EV（Earned Value，挣值）

EV（Earned Value，挣值）表示截至某一时点，实际已完成工作的预算价值。

可以简单理解为：

- 实际已经挣到多少“预算价值”

### 3. AC（Actual Cost，实际成本）

AC（Actual Cost，实际成本）表示截至某一时点，项目真实花出去的成本。

可以简单理解为：

- 实际已经花了多少钱

### 4. 为什么这三个量必须放在一起看

只看 AC，你只知道“花了多少”；

只看 EV，你只知道“做出来了多少预算价值”；

只看 PV，你只知道“按计划本该做到哪里”。

只有把三者放在一起，才能同时判断：

- 是否超支
- 是否拖期
- 做了多少值钱的工作

## 6. 偏差与绩效指数

### 1. 成本偏差 CV（Cost Variance）

- CV = EV - AC

判断规则：

- CV > 0：成本节约
- CV = 0：成本正常
- CV < 0：成本超支

### 2. 进度偏差 SV（Schedule Variance）

- SV = EV - PV

判断规则：

- SV > 0：进度超前
- SV = 0：进度正常
- SV < 0：进度落后

### 3. 成本绩效指数 CPI（Cost Performance Index）

- CPI = EV / AC

判断规则：

- CPI > 1：成本绩效好，单位成本挣回的价值更多
- CPI = 1：成本正常
- CPI < 1：成本绩效差，存在超支

### 4. 进度绩效指数 SPI（Schedule Performance Index）

- SPI = EV / PV

判断规则：

- SPI > 1：进度超前
- SPI = 1：进度正常
- SPI < 1：进度落后

### 5. 最容易错的地方

- 把 CPI 和 SPI 的分子分母写反
- 把 CV、SV 的正负号含义记反
- 把 EV 误看成 AC 或 PV

## 7. 完工估算与后续预测指标

### 1. BAC（Budget at Completion，完工预算）

BAC（Budget at Completion，完工预算）表示项目完工预算，也就是项目原计划总预算。

### 2. EAC（Estimate at Completion，完工估算）

EAC（Estimate at Completion，完工估算）表示完工估算，即按当前情况预测项目最终总成本。

常见公式有三类：

#### 1. 若认为当前成本绩效会持续

- EAC = BAC / CPI

#### 2. 若认为后续工作可按原预算效率完成

- EAC = AC + (BAC - EV)

#### 3. 若同时考虑成本和进度效率的持续影响

- EAC = AC + (BAC - EV) / (CPI × SPI)

### 3. ETC（Estimate to Complete，尚需完工估算）

ETC（Estimate to Complete，尚需完工估算）常见理解为：

- ETC = EAC - AC

### 4. VAC（Variance at Completion，完工偏差）

VAC（Variance at Completion，完工偏差）常见理解为：

- VAC = BAC - EAC

判断规则：

- VAC > 0：预计最终低于预算，有结余
- VAC < 0：预计最终超过预算，有超支

## 8. 做挣值题时的推荐顺序

看到挣值题，不要急着套公式，建议按下面顺序做：

1. 先圈出已知量：PV、EV、AC、BAC。
2. 判断题目问的是偏差、指数，还是最终预测。
3. 先算 CV、SV，再算 CPI、SPI。
4. 再根据题目条件选择 EAC 公式。
5. 最后把计算结果翻译成管理语言，例如“超支但进度超前”。

这一步非常关键，因为考试往往不是只问你算多少，而是要你判断：

- 当前项目状态如何
- 偏差可能由什么引起
- 项目经理接下来应该做什么

## 9. 成本失控常见根因

项目成本失控很少是单一原因，常见根因包括：

- 需求频繁变化导致范围扩大
- 估算过粗，前期低估工作量
- 外部采购价格波动或供应商履约差
- 返工和质量缺陷过多
- 关键资源冲突导致加班和赶工成本上升
- 没有形成成本基准或没有按基准监控
- 只记实际支出，不做挣值分析

## 10. 成本管理和其他知识域的关系

成本管理不能孤立看，它和其他知识域高度耦合：

- 范围变化会直接改变成本
- 进度延期通常会增加资源投入和管理成本
- 质量缺陷和返工会增加 AC
- 采购和合同管理会影响外部成本支出结构
- 风险事件会消耗应急储备

所以写案例题和论文时，不要只写“控制成本”，而要写清：

- 范围怎么冻结
- 进度怎么监控
- 供应商怎么管
- 质量怎么预防返工
- 风险怎么提前准备储备

## 11. 今天最容易混的概念

| 易混概念 | 正确区分 |
| --- | --- |
| 成本估算 vs 制定预算 | 一个偏测算，一个偏汇总形成基准 |
| 成本基准 vs 项目预算 | 一个偏控制基准，一个偏总资金安排 |
| PV vs EV vs AC | 一个计划价值，一个挣值，一个实际成本 |
| CV vs SV | 一个看成本偏差，一个看进度偏差 |
| CPI vs SPI | 一个看成本效率，一个看进度效率 |
| 应急储备 vs 管理储备 | 一个应对已识别风险，一个应对未识别风险 |
| 控制成本 vs 单纯压缩费用 | 一个强调基于基准监控和纠偏，一个可能只是粗暴削减投入 |

答案：C

## 八、案例题

### 1. 案例背景

某市智慧园区综合管理平台项目总预算 BAC 为 980 万元，计划建设周期 14 个月。项目进行到第 8 个月末时，项目经理组织阶段性成本分析，得到以下数据：

- PV = 560 万元
- EV = 500 万元
- AC = 620 万元

同时，项目还出现了以下情况：

- 建设单位口头要求新增移动巡检细化功能，但未完成正式变更审批。
- 某第三方硬件供应商因交货延期，导致现场实施多次重复进场。
- 项目组前期估算较粗，对接口联调和现场实施复杂度预估不足。
- 月度例会只统计实际花费，没有定期做挣值分析。
- 为追赶里程碑，团队安排了较多加班和临时外包支持。

### 2. 问题

1. 请计算该项目当前的 CV、SV、CPI、SPI，并判断项目当前成本和进度状态。
2. 若认为当前成本绩效会持续，试估算 EAC。
3. 结合案例，指出至少 5 项项目在成本管理方面存在的问题。
4. 站在项目经理角度，给出至少 6 条改进措施。

### 3. 参考答案要点

#### 第 1 问：计算与判断

- CV = EV - AC = 500 - 620 = -120 万元
- SV = EV - PV = 500 - 560 = -60 万元
- CPI = EV / AC = 500 / 620 ≈ 0.81
- SPI = EV / PV = 500 / 560 ≈ 0.89

判断：

- CV < 0，说明项目成本超支
- SV < 0，说明项目进度落后
- CPI < 1，说明成本绩效差
- SPI < 1，说明进度绩效差

因此，该项目当前处于“超支且落后”的状态。

#### 第 2 问：EAC 估算

若认为当前成本绩效将持续，则：

- EAC = BAC / CPI = 980 / 0.81 ≈ 1209.88 万元

可写为：

- 预计完工总成本约为 1210 万元

这意味着项目若不采取措施，最终可能超预算约 230 万元左右。

#### 第 3 问：存在的问题

可从以下角度作答：

1. 新增需求未经过正式变更审批，范围变化未同步到成本控制体系中。
2. 前期估算过粗，对接口联调、现场实施和供应商协同复杂度评估不足。
3. 未建立或未严格执行按阶段、按工作包的成本监控机制。
4. 月度管理只看实际花费，没有结合 EV、PV、AC 进行综合分析。
5. 供应商履约风险未被有效控制，导致重复进场和额外实施成本增加。
6. 为追赶进度大量加班和临时外包，抬高了实际成本 AC。
7. 进度问题与成本问题联动明显，但没有及时采取一体化纠偏措施。

#### 第 4 问：改进措施

可从以下角度组织答案：

1. 重新梳理 WBS 和成本估算，针对高风险工作包做更细粒度测算。
2. 严格执行变更控制流程，对新增需求进行范围、进度、成本影响分析后再决策。
3. 以成本基准为依据，建立按月或按里程碑的挣值分析机制。
4. 对供应商交付、采购价格和现场实施安排加强跟踪，降低重复进场和等待成本。
5. 对加班、外包、返工等高成本活动做专项分析，避免无效投入持续扩大。
6. 将成本控制与进度控制联动起来，优先处理关键路径上既拖期又高成本的工作。
7. 合理使用应急储备，对已识别风险造成的额外支出进行规范化管理。
8. 及时向建设单位和关键干系人报告偏差状态，必要时发起正式调整或预算变更。
