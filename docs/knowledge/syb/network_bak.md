## 二、计算机网络是什么

### 1. 计算机网络的定义

计算机网络是由若干计算机系统和网络设备，通过通信链路连接起来，并依照约定的通信协议实现数据交换与资源共享的系统。

更直白地说，网络解决的是“不同位置、不同设备、不同系统之间如何可靠地传递信息”的问题。

网络不仅仅是让两台机器连通，它还要解决：

- 数据怎么组织与传输
- 路径怎么选择
- 差错怎么处理
- 速度怎么匹配
- 安全怎么保障

### 2. 计算机网络与互联网、通信系统、分布式系统的关系

#### 与互联网的关系

互联网是目前全球范围最大、最复杂的计算机网络。学习计算机网络，实际上是在学习互联网背后的运行规律。

#### 与通信系统的关系

通信系统是更宽泛的概念，电话、广播、卫星通信都属于通信系统。计算机网络是其中面向数字数据交换的重要分支。

#### 与分布式系统的关系

分布式系统依赖网络完成节点间通信。没有网络，分布式系统无法运行。网络质量通常直接影响分布式系统的稳定性和性能。

### 3. 学习计算机网络在软件工程中的意义

对于开发者来说，网络不是“运维才需要懂”的知识。很多线上问题都和网络相关，例如：

- 接口超时
- DNS 解析失败
- TLS 证书异常
- 连接池耗尽
- 跨地域访问延迟高

懂网络，能帮助你更快定位问题，也能帮助你设计更稳健的系统架构。

## 三、为什么需要计算机网络

### 1. 资源共享

网络让多台主机共享数据与服务，例如：

- 文件共享
- 打印服务
- 数据库服务
- 缓存服务

### 2. 协作通信

现代软件系统几乎都依赖网络协作：

- 前后端通信
- 微服务调用
- 消息队列投递
- 第三方 API 对接

### 3. 规模化运行

当系统从单机变成集群，网络就成为基础能力。负载均衡、故障切换、服务发现，都建立在网络互联之上。

### 4. 远程访问

企业办公、云服务、移动互联网、物联网，都依赖跨地域网络连接能力。

## 四、计算机网络的核心组成

### 1. 终端与主机

终端和主机是网络中的数据源和数据目的地。

例如：

- 个人电脑
- 手机
- 服务器
- IoT 设备

### 2. 网络接口

网卡负责把主机的数据转换为可以在链路上传输的信号。

常见参数：

- MAC 地址
- 速率（1Gbps/10Gbps）
- 双工模式

### 3. 传输介质

包括双绞线、光纤、无线信道等。不同介质在成本、距离、带宽和抗干扰能力上差异明显。

### 4. 网络设备

- 交换机：二层转发，基于 MAC 地址
- 路由器：三层转发，基于 IP 地址
- 防火墙：访问控制与安全防护
- 负载均衡：流量分发与故障剔除

### 5. 协议

协议是通信规则。没有统一协议，不同设备无法理解彼此传输的数据。

## 五、网络分层思想

### 1. 为什么要分层

分层的核心价值是复杂问题分解。

好处包括：

- 降低复杂度
- 便于标准化
- 便于替换某一层实现
- 便于排障定位

### 2. OSI 七层模型

```text
7. 应用层
6. 表示层
5. 会话层
4. 传输层
3. 网络层
2. 数据链路层
1. 物理层
```

OSI 模型更偏理论教学，但它能帮助理解不同层的职责边界。

### 3. TCP/IP 四层模型

```text
应用层
传输层
网际层
网络接口层
```

工程实践中更常使用 TCP/IP 模型。

### 4. 封装与解封装

发送端：应用数据逐层封装头部信息后发送。

接收端：逐层解封装还原成应用数据。

```text
应用数据 -> [TCP头+数据] -> [IP头+TCP段] -> [MAC头+IP包] -> 比特流
```

## 六、物理层与数据链路层

### 1. 物理层

物理层处理比特流传输，定义接口、电气特性、传输速率、编码方式等。

关键概念：

- 带宽
- 信道
- 调制
- 误码率

### 2. 数据链路层

数据链路层把网络层的数据包封装为帧，并在同一链路范围内传输。

关键功能：

- 成帧
- 差错检测
- 介质访问控制

### 3. MAC 地址

MAC 地址是网卡的硬件地址，理论上全球唯一，常用 48 位表示。

示例：`00:1A:2B:3C:4D:5E`

### 4. 交换机与二层转发

交换机维护 MAC 地址表，学习端口与 MAC 的映射关系，再进行帧转发。

简化逻辑：

- 目标 MAC 已知：定向转发
- 目标 MAC 未知：泛洪

### 5. VLAN

VLAN 用于划分二层广播域。即使在同一交换机上，不同 VLAN 的主机也默认二层隔离。

示意图：

```text
[PC-A VLAN10] --\
                [Switch] --- [Router/L3Switch] --- [PC-C VLAN20]
[PC-B VLAN10] --/
```

### 6. ARP 协议

ARP 用于通过 IP 地址查询目标 MAC 地址。

典型过程：

1. 主机广播 ARP 请求
2. 目标主机单播 ARP 响应
3. 请求方缓存 IP-MAC 映射

常见问题：ARP 欺骗。

## 七、网络层

### 1. IP 地址

IP 地址用于网络层寻址。

IPv4 示例：`192.168.1.10`

IPv6 示例：`2001:db8::10`

### 2. 子网与 CIDR

CIDR 表示法示例：`192.168.1.0/24`

其中 `/24` 表示网络前缀长度。

常见换算：

- `/24` 子网掩码：255.255.255.0
- `/16` 子网掩码：255.255.0.0

### 3. 路由与转发

路由器根据路由表选择下一跳，完成跨网段通信。

路由来源：

- 直连路由
- 静态路由
- 动态路由

### 4. ICMP

ICMP 常用于网络诊断。

典型命令：

```bash
ping 8.8.8.8
traceroute example.com
```

### 5. NAT

NAT 用于私网地址与公网地址转换，缓解 IPv4 地址不足并提升一定隔离性。

常见类型：

- SNAT
- DNAT
- PAT

### 6. IPv4 与 IPv6

IPv6 优势：

- 地址空间更大
- 更好的自动配置能力
- 去除了广播机制，改为多播
- 内置对扩展和安全机制更友好

## 八、传输层

### 1. TCP 与 UDP

| 维度 | TCP | UDP |
|------|-----|-----|
| 连接方式 | 面向连接 | 无连接 |
| 可靠性 | 可靠传输 | 尽力而为 |
| 顺序性 | 保证顺序 | 不保证顺序 |
| 开销 | 较大 | 较小 |
| 典型场景 | Web、数据库 | 实时音视频、DNS |

### 2. 端口与套接字

端口用于标识主机上的应用进程。

- 0-1023：知名端口
- 1024-49151：注册端口
- 49152-65535：动态端口

套接字可简化理解为：`IP + 端口 + 协议`。

### 3. TCP 三次握手

```text
客户端 -> SYN -> 服务端
客户端 <- SYN+ACK <- 服务端
客户端 -> ACK -> 服务端
```

目的：

- 确认双方收发能力
- 协商初始序列号

### 4. TCP 四次挥手

```text
客户端 -> FIN -> 服务端
客户端 <- ACK <- 服务端
客户端 <- FIN <- 服务端
客户端 -> ACK -> 服务端
```

### 5. 流量控制

流量控制解决发送方速度过快导致接收方处理不过来的问题。TCP 通过滑动窗口机制实现。

### 6. 拥塞控制

拥塞控制关注的是整个网络是否拥堵。

常见机制：

- 慢启动
- 拥塞避免
- 快重传
- 快恢复

### 7. 常见传输层问题

- 建连超时
- 连接被重置
- 大量 TIME_WAIT
- 重传率高
- 半连接堆积

## 九、应用层协议

### 1. DNS

DNS 把域名解析为 IP 地址。

简化流程：

```text
浏览器 -> 本地DNS缓存 -> 递归DNS服务器 -> 权威DNS服务器 -> 返回IP
```

常见记录：

- A
- AAAA
- CNAME
- MX
- TXT

### 2. HTTP 与 HTTPS

HTTP 是应用层协议，HTTPS 是 HTTP + TLS。

HTTP 常见方法：

- GET
- POST
- PUT
- DELETE
- PATCH

HTTP 状态码示例：

- 200 成功
- 301/302 重定向
- 400 请求错误
- 401 未认证
- 403 禁止访问
- 404 资源不存在
- 500 服务端错误

HTTPS 增强点：

- 传输加密
- 身份认证
- 完整性校验

### 3. 邮件协议

- SMTP：发邮件
- POP3：拉取邮件，偏下载
- IMAP：邮件同步管理

### 4. DHCP

DHCP 自动分配 IP、网关、DNS 等网络参数。

典型过程：DORA

```text
Discover -> Offer -> Request -> Acknowledge
```

### 5. SSH

SSH 提供加密远程登录能力，常用于服务器管理。

### 6. WebSocket

WebSocket 支持全双工、长连接，适合实时通信场景，如在线聊天和协同编辑。

## 十、网络性能与质量指标

### 1. 延迟

延迟是数据从发送到接收所花费时间。

常见分解：

- 传播延迟
- 传输延迟
- 排队延迟
- 处理延迟

### 2. 抖动

抖动是延迟波动程度，对音视频体验影响明显。

### 3. 带宽与吞吐

- 带宽：链路理论最大能力
- 吞吐：实际传输速率

### 4. 丢包

丢包会触发重传，导致延迟上升和吞吐下降。

### 5. 可用性

可用性常用“几个 9”表示。

示例：

- 99.9%（年不可用约 8.76 小时）
- 99.99%（年不可用约 52.6 分钟）

### 6. 性能观测实践

常见手段：

- 主机指标监控
- 网络设备监控
- 应用层链路追踪
- 合成探测
- 抓包分析

## 十一、网络安全基础

### 1. 常见攻击类型

- DDoS
- 中间人攻击
- ARP 欺骗
- DNS 污染
- SQL 注入
- 会话劫持

### 2. 加密基础

- 对称加密：加解密速度快
- 非对称加密：用于密钥交换和签名
- 哈希：用于完整性校验

### 3. TLS 证书

证书用于证明服务端身份，避免连接到伪造站点。

### 4. 防火墙与访问控制

防火墙规则通常基于五元组：

- 源 IP
- 目的 IP
- 源端口
- 目的端口
- 协议

### 5. 零信任与最小权限

零信任强调“默认不信任，持续验证”。

最小权限强调“只给完成任务所需的最小访问权限”。

### 6. 安全运维

- 定期漏洞扫描
- 证书与密钥轮转
- 安全日志审计
- 异常流量告警

### 4. 云网络 VPC

VPC 可理解为云上的逻辑隔离网络。

核心组件：

- 子网

- 安全组

### 5. 服务网格

服务网格用于管理微服务间通信，常见能力：

- 流量治理
- 可观测性
- 安全策略

## 十三、网络排障方法

### 1. 分层排障思路

优先确定问题在哪一层：

- 物理层：链路是否通
- 二层：VLAN、MAC、ARP 是否正常
- 三层：IP、路由、NAT 是否正常
- 四层：端口、连接状态是否正常
- 七层：协议、证书、应用逻辑是否正常

### 2. 常见命令

```bash
ip addr
ip route
ping <target>
traceroute <target>
nslookup <domain>
dig <domain>
ss -ant
netstat -ant
curl -v https://example.com
```

### 3. 抓包基础

```bash
tcpdump -i any host 10.0.0.5 and port 443
```

抓包关注点：

- 是否发出请求
- 是否收到响应
- 是否有重传
- TCP 握手是否完成
- TLS 握手在哪一步失败

### 4. 典型故障案例

#### 案例 A：域名可解析但接口超时

排查路径：

1. DNS 正常
2. 检查安全组端口是否放行
3. 检查服务监听地址是否为 0.0.0.0
4. 检查上游负载均衡健康检查状态

#### 案例 B：偶发 502

排查路径：

1. 确认网关与后端连接池配置
2. 查看后端超时阈值
3. 检查后端实例是否频繁重启
4. 查看流量峰值时段的错误分布

## 十五、关键概念示例与图例

### 1. TCP 建连时序图

```text
Client                                  Server
  | --- SYN(seq=x) --------------------> |
  | <--- SYN+ACK(seq=y, ack=x+1) ------- |
  | --- ACK(ack=y+1) ------------------> |
  |          连接建立完成                |
```

### 2. HTTP 请求链路示例

```text
浏览器
  -> 本地 DNS
  -> 递归 DNS
  -> 目标 IP
  -> CDN / 负载均衡
  -> 应用网关
  -> 应用服务
  -> 数据库/缓存
```

### 3. 子网划分示例

给定网络 `192.168.10.0/24`，若划分为 4 个子网，可得到：

- 192.168.10.0/26
- 192.168.10.64/26
- 192.168.10.128/26
- 192.168.10.192/26

每个子网可用主机数：62。

### 4. Python Socket 示例

```python
import socket


def simple_tcp_client(host: str, port: int, message: str) -> str:
    with socket.create_connection((host, port), timeout=3) as conn:
        conn.sendall(message.encode("utf-8"))
        data = conn.recv(4096)
        return data.decode("utf-8", errors="ignore")


if __name__ == "__main__":
    # 示例：连接本地 echo 服务
    try:
        response = simple_tcp_client("127.0.0.1", 9000, "hello")
        print("response:", response)
    except Exception as exc:
        print("network error:", exc)
```

## 十六、子知识点深入讲解

### 1. DNS 深入

#### 1.1 递归解析与迭代解析

- 递归解析：客户端把问题交给递归 DNS，由它负责最终答案。
- 迭代解析：DNS 服务器逐步返回“下一步去问谁”的信息。

#### 1.2 缓存与 TTL

TTL 决定记录在缓存中可保留多久。TTL 太长会导致变更生效慢，TTL 太短会增加解析压力。

#### 1.3 DNS 常见故障

- 缓存污染
- 记录配置错误
- 权威 DNS 不可用
- DNSSEC 配置异常

### 2. HTTP 深入

#### 2.1 HTTP/1.1

特点：

- 文本协议
- 支持持久连接
- 存在队头阻塞问题

#### 2.2 HTTP/2

特点：

- 二进制分帧
- 多路复用
- 头部压缩

#### 2.3 HTTP/3

基于 QUIC（UDP），优化握手时延与丢包场景表现。

### 3. TLS 深入

#### 3.1 TLS 握手目标

- 协商算法
- 验证身份
- 安全交换密钥
- 建立加密通信

#### 3.2 证书校验要点

- 证书链是否完整
- 证书是否过期
- 域名是否匹配
- 根证书是否可信

### 4. 路由协议简述

#### 4.1 OSPF

- 链路状态路由协议
- 适合中大型内部网络

#### 4.2 BGP

- 路径向量协议
- 互联网自治系统间路由核心协议

### 5. 负载均衡深入

#### 5.1 四层负载均衡

基于 IP+端口转发，不解析应用层内容，性能高。

#### 5.2 七层负载均衡

可基于 URL、Header、Cookie 等做精细路由，灵活但开销更高。

#### 5.3 常见策略

- 轮询
- 最少连接
- 一致性哈希
- 加权分发

### 6. NAT 深入

#### 6.1 SNAT

源地址转换，常用于私网访问公网。

#### 6.2 DNAT

目的地址转换，常用于公网访问私网服务。

#### 6.3 NAT 带来的影响

- 连接追踪开销
- 某些协议穿透困难
- 日志审计复杂度上升

### 7. 云原生网络深入

#### 7.1 容器网络

容器网络核心问题：

- Pod/容器如何获取 IP
- 容器间如何跨节点通信
- 服务发现如何实现

#### 7.2 CNI

CNI 是容器网络接口标准，常见实现包括 Calico、Flannel、Cilium。

#### 7.3 Ingress 与网关

Ingress 负责集群外到集群内服务的入口管理。

### 8. 网络可靠性深入

#### 8.1 故障域

设计时应尽量限制故障影响范围，避免单点故障放大。

#### 8.2 冗余

- 设备冗余
- 链路冗余
- 地域冗余

#### 8.3 演练

定期故障演练有助于验证预案有效性。

## 十八、常见面试/考试型问答示例

### 1. 为什么 TCP 需要三次握手而不是两次

两次握手无法可靠确认双方收发能力和初始序列号状态，可能导致历史连接请求引发误判。三次握手可以建立更可靠的连接上下文。

### 2. 为什么挥手需要四次

TCP 是全双工通信，双方关闭发送方向通常需要分别发送 FIN 与 ACK，因此常见为四次。

### 3. HTTPS 一定绝对安全吗

不是。HTTPS 解决的是传输加密与身份认证问题，但不能替代业务层安全设计，例如鉴权、输入校验、权限控制。

### 4. ping 通但业务不通可能的原因

- 端口未开放
- 防火墙拦截
- 应用未监听
- 证书错误
- 应用层协议不匹配

### 5. 访问慢如何定位

建议分层定位：

1. DNS 是否慢
2. 建连是否慢
3. TLS 握手是否慢
4. 服务端处理是否慢
5. 返回包是否出现重传

## 二十、总结

计算机网络不是单一协议的堆砌，而是一整套分层协作系统。

要建立完整概念，需要抓住四条主线：

- 结构主线：从物理层到应用层的职责分工
- 协议主线：IP、TCP、DNS、HTTP、TLS 等如何协作
- 工程主线：高可用、可扩展、可观测、安全治理
- 排障主线：分层定位、证据驱动、逐步收敛

如果继续深入，建议沿着“协议原理 + 抓包实践 + 生产案例复盘”的路径学习。真正掌握网络，不只是记住定义，而是能够在真实问题中快速判断和定位。
